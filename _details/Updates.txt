Update 3: Engine Improvements
10. Autoplay: Addresses the most common complaints users have with native YouTube Music or Spotify autoplay: The dreaded "Artist Loop."
If a queue runs dry on a track by 'Bad Omens', standard autoplay algorithms often get lazy and just feed four more 'Bad Omens' tracks in a row. By implementing a smart artist shuffler, the bot lets the queue breathe. It ensures that the heavy, modern metalcore energy continues by gracefully transitioning to 'Spiritbox', then 'Architects', and maybe 'Annisokay', giving you a dynamic radio experience without compromising the core vibe.
This is how the smart shuffling algorithm works into the current architecture:
1. The Artist History: Attach a lightweight array to Shoukaku player object to track the last N artists played (e.g., keeping a rolling memory of the last 3 to 5 artists).
2. The Recommendation Pool: Instead of blindly grabbing the #1 recommended song from ytmusic-api, fetch the top 15 or 20 "Up Next" tracks.
3. The Filter: Loop through that pool of 20 tracks. Pick the highest-ranked track whose artist does not exist in the Artist History array.
4. The Rotation: Once a song is picked, push its artist to the Artist History array. If the array gets larger than the limit (e.g., 5), remove the oldest artist.
- Task 10.1: Track the Artist History (/src/lavalink/trackStart.js)
    - Initialize the memory array on the player and update it every time a new song starts.
- Task 10.2: Update the YTMusic Service (/src/services/ytmusic.js)
    - Modify the recommendation function to accept the artistHistory array and filter the results.
- Task 10.3: Trigger the Smart Autoplay (/src/lavalink/trackEnd.js)
    - When the queue ends, pass the last played track's ID and the player's artist history into the new smart function.

11. Regular Expression (Regex) Blacklist: Because ytmusic-api pulls data directly from YouTube, uploaders use very predictable naming conventions for modified audio (e.g., putting modifiers in parentheses like (Slowed + Reverb) or after a dash like - Acoustic). By compiling these common keywords into a single Regex pattern, every track title in the recommendation pool needed to be instantly tested and discard the unofficial ones.
- Task 11.1: Define the Regex Blacklist
    - create a comprehensive regex pattern that targets modifiers like (Slowed + Reverb), - Acoustic, etc. plus a few other common ones that plague YouTube Music recommendations.
    - The i flag at the end of the regex makes it case-insensitive, meaning it will catch "Remix", "REMIX", and "remix" equally.
- Task 11.2: Filter the Recommendations (/src/services/ytmusic.js)
    - Update the ytmusic.js service to evaluate both the artist history and the title string before accepting a track.
