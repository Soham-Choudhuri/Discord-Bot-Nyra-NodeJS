1. Core Intelligence & Fetching: Instead of blindly picking "related" videos, the system hooks directly into the YouTube Music (YTM) native algorithm.
- Micro-Batching: It only requests 25 tracks at a time. This keeps the memory footprint extremely light and ensures the recommendations remain tightly relevant to the immediate context rather than fetching hundreds of irrelevant songs.
- Redundancy & Reliability: The fetch loop is wrapped in a concurrent executor with a 3-attempt exponential backoff retry system, ensuring the music never stops even if the YouTube API temporarily rate-limits the bot.

2. Advanced Heuristics & Quality Enforcement: The raw data from YouTube is often dirty (filled with fan-edits, live performances, or music videos with 2-minute dialogue intros).
- Audio Track Video (ATV) Isolation: It strictly enforces an ALLOWED_TYPES = ['MUSIC_VIDEO_TYPE_ATV'] rule. Official Music Videos (OMVs) and User Generated Content (UGC) are instantly rejected. This guarantees the bot only streams pure, studio-quality audio tracks without background noise or dialogue.
- Modification Banlist: It runs a strict string-matching filter across titles and artist metadata, outright rejecting any track containing the words: "slowed", "reverb", "nightcore", "sped up", "chipmunk", etc.

3. Seed Algorithms (Dynamic vs Static): The buffer filling system will support two distinct mathematical approaches to generating the radio queue:
- Dynamic Mode: The generation seed constantly evolves. Every time the buffer runs low, it uses the last track in the buffer to generate the next batch. This allows the genre to naturally and fluidly drift over time.
- Static Mode: The seed is permanently locked to the very first song you played. Every future batch of tracks is generated directly against that original song, enforcing a perfectly consistent "vibe" that never drifts.

4. State Memory & Shuffling
- Infinite Memory: A variable keeps track of the ID of every single song played during the session. The buffer filling logic cross-references new API data against this history, guaranteeing zero repeats.
- Smart Artist Interleaving: Instead of randomizing the buffer natively (which can cause clumping where 3 songs from the same artist play back-to-back), use a Smart Shuffle algorithm. This groups the tracks by artist and deals them out like a deck of cards, guaranteeing maximum artist variety.

5. Just-In-Time (JIT) Resolution
- Strict Hardware Isolation: To prevent CPU spikes that cause audio stuttering, the buffer must not aggressively download metadata. It only holds the raw JSON. The actual Wavelink track resolution happens strictly inline when the current track finishes.
