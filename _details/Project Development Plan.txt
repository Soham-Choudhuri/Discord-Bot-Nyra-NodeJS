1. Tech Stack Overview
- Runtime: Node.js (v18.x or higher recommended)
- Discord Library: discord.js (v14)
- Lavalink Client: shoukaku (Highly stable, handles Lavalink v4 flawlessly)
- YouTube Music Data: ytmusic-api (NPM equivalent of Python's ytmusicapi)
- Environment Manager: dotenv

2. Project Initialization & Directory Structure
- Before writing code, establish a scalable, modular architecture.
    - Task 2.1: Initialize the project
        - Run npm init -y in your project folder.
        - Install core dependencies: npm install discord.js shoukaku ytmusic-api dotenv
        - Install development dependencies (optional but recommended): npm install nodemon --save-dev
    - Task 2.2: Create the Modular Folder Structure

3. Configuration & Environment Setup
    - Task 3.1: Configure .env
        DISCORD_TOKEN=your_bot_token_here
        CLIENT_ID=your_bot_client_id
        LAVALINK_HOST=localhost
        LAVALINK_PORT=2333
        LAVALINK_PASSWORD=youshallnotpass

4. Core Bot Setup & Handlers
    - Task 4.1: Build the Command & Event Handlers (/src/handlers/)
        Create modular loaders that dynamically read files from the /commands and /events folders.
        Command Handler: Reads all .js files in /commands and stores them in a Discord.Collection(). It also registers these as Slash Commands with Discord's REST API.
        Event Handler: Reads files in /events and attaches them to client.on() or client.once().
    - Task 4.2: Initialize the Main Entry Point (/src/index.js)
        Instantiate the discord.js Client with necessary intents (Guilds, GuildVoiceStates). Load your handlers and login using the token from your .env file.

5. Lavalink Implementation (Shoukaku)
    - Task 5.1: Initialize Shoukaku (/src/index.js or a dedicated wrapper)
        Configure the Shoukaku client and attach it to your Discord client.
    - Task 5.2: Lavalink Event Listeners (/src/lavalink/)
        Create listeners for Shoukaku events to handle node stability and track playback:
            - ready: Log when Lavalink connects successfully.
            - error: Handle WebSocket disconnects or Lavalink crashes gracefully.
            - trackStart: Send a message to the Discord channel when a song begins.
            - trackEnd: Logic to play the next song in the queue or destroy the player if the queue is empty.

6. Integrating ytmusic-api (The Python Equivalent)
    - Task 6.1: Create the YTMusic Service (/src/services/ytmusic.js)
        Initialize the YTMusic instance.
        Create utility functions for: searching, fetching video details, getting playlist tracks, and fetching "Next" recommendations.

7. Building the Slash Commands
    - Task 7.1: Build /stream command
        Validation: Check if the user is in a Voice Channel.
        Search: Accept a query or URL. Use Shoukaku's REST API (client.manager.getNode().rest.resolve(query)) to search Lavalink. Because you have LavaSrc, prefix searches with ytmsearch: to force YouTube Music tracking.
        Connection: Join the voice channel using client.manager.joinVoiceChannel().
        Playback: Add the returned track to your custom Queue array and call .playTrack() on the Shoukaku player.
    - Task 7.2: Build /autoplay command (Using ytmusic-api)
        When a queue ends, instead of leaving the channel, grab the ID of the last played track.
        Pass that ID to your ytmusic-api service (getAdvancedRecommendations).
        Fetch the YouTube Music algorithm's top recommended track.
        Pass that track name back into Lavalink to play, simulating native YouTube Music autoplay.
    - Task 7.3: Basic Control Commands
        /skip: Call .stopTrack() on Shoukaku (which triggers the trackEnd event to play the next song).
        /stop: Destroy the Shoukaku player and clear the queue.
        /queue: Display the current array of saved tracks.
        /leave: Disconnects the bot from the voice channel.

8. Testing & Edge Case Handling
    - Task 8.1: Voice State Updates
        Listen to Discord's voiceStateUpdate event. If the bot is left alone in a voice channel, trigger a timeout to automatically disconnect and save resources.
    - Task 8.2: Rate Limits
        Even with plugins, YouTube can throttle IPs. If Lavalink throws a 429 error, ensure your bot gracefully informs the user rather than crashing.
